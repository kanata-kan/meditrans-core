🔴 الجزء A — محرك PDF (قابل لإعادة الاستخدام)

محرك مستقل تماماً مبني بـ @react-pdf/renderer. يقبل بيانات عامة (InvoicePdfData) وتكوين (InvoicePdfConfig) — لا يعرف شيئاً عن Prisma أو MediTrans.

src/modules/invoices/pdf/ ├── invoicePdf.types.ts ← أنواع عامة: Config + Data + Line + Client ├── invoicePdf.config.ts ← تكوين MediTrans الافتراضي + mergeConfig() ├── InvoiceDocument.tsx ← Document root + metadata (title, author) ├── InvoiceTemplate.tsx ← A4 layout يجمع كل المكونات + ترقيم ├── InvoiceHeader.tsx ← اسم الشركة + هاتف + إيميل + ICE ├── InvoiceClientSection.tsx ← رقم الفاتورة + تاريخ + معلومات العميل ├── InvoiceTable.tsx ← ⭐ جدول multi-page آمن (wrap + fixed header) ├── InvoiceTotals.tsx ← HT + TVA + TTC + المبلغ المدفوع/المتبقي ├── InvoiceFooter.tsx ← ملاحظات + نص قانوني (fixed position) └── generateInvoicePdf.tsx ← ⭐ renderToStream → Buffer (Vercel safe)
⚙️ InvoicePdfConfig
companyName: اسم الشركة
companyAddress/Phone/Email: بيانات الاتصال
companyTaxId: الرقم الضريبي (ICE)
primaryColor / secondaryColor: ألوان العلامة التجارية
currency: MAD | EUR | USD
locale: fr-MA لتنسيق الأرقام
labels: كل التسميات قابلة للتخصيص
footerText: نص قانوني اختياري
📊 InvoicePdfData
invoiceNumber: رقم الفاتورة
issueDate / dueDate: تاريخ الإصدار والاستحقاق
client: { name, phone, email, address }
patientName: اسم المريض (اختياري)
lines[]: { label, quantity, unitPriceHt, totalHt, tvaRate, totalTtc }
subtotalHt / totalTva / totalTtc: المجاميع
amountPaid: المبلغ المدفوع (لفواتير جزئية)

// الاستخدام — بسيط وواضح import { generateInvoicePdf } from "@/modules/invoices/pdf/generateInvoicePdf"; // 1. بالتكوين الافتراضي (MediTrans) const buffer = await generateInvoicePdf(data); // 2. بتكوين مخصص const buffer = await generateInvoicePdf(data, { companyName: "My Company", currency: "EUR", primaryColor: "#e74c3c", }); // المولد يستخدم renderToStream → Buffer // متوافق مع Vercel serverless (لا puppeteer، لا ملفات مؤقتة)

🌐 API Route — تحميل PDF

📌 المسار: GET /api/invoices/[id]/pdf
📌 الآلية: يجلب الفاتورة → يحول Prisma → InvoicePdfData → يولد PDF → يرجع binary
📌 Content-Type: application/pdf
📌 Content-Disposition: inline (عرض في المتصفح مباشرة)
📌 اسم الملف: INV-2025-0001.pdf تلقائياً
📌 Cache: no-store — كل طلب يولد PDF جديد

⚡ Server Actions

📌 createInvoiceAction(rawInput) — إنشاء فاتورة من مجموعة خدمات + revalidatePath
📌 cancelInvoiceAction(id) — إلغاء (مرفوض إذا فيها دفعات)
📌 listInvoicesAction(filters?) — جلب الكل مع العميل والخطوط والدفعات
📌 getInvoiceAction(id) — فاتورة واحدة بكل التفاصيل
📌 getInvoicePdfDataAction(id) — الجسر: Prisma → InvoicePdfData (Decimal→number)
📌 loadInvoiceFormDataAction() — قائمة العملاء لفورم الإنشاء
📌 loadUninvoicedServicesAction(clientId) — خدمات غير مفوترة مع snapshots

🏗️ القرارات المعمارية

📌 فصل كامل PDF ↔ UI: مكونات PDF لا تستورد أي شيء من React UI والعكس
📌 محرك عام 100%: لا import من Prisma ولا MediTrans — فقط types عامة
📌 API Route وليس Server Action: PDF يحتاج Response مخصص (binary + content-type)
📌 Buffer وليس Stream: متوافق مع Vercel serverless
📌 جسر واحد: getInvoicePdfDataAction هو المكان الوحيد الذي يحول Decimal→number
📌 mergeConfig(): deep merge يسمح بتخصيص جزئي بدون فقدان الافتراضيات
📌 wrap + fixed: الجدول يتقسم تلقائياً على عدة صفحات بدون تداخل
📌 قابل للتوسيع: QR code · توقيع رقمي · badge حالة الدفع · متعدد اللغات — كلها ممكنة بدون إعادة كتابة
