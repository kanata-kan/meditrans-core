✏️ ServiceForm — الفورم الذكي

فورم إنشاء الخدمة هو أعقد مكون في المشروع — يربط بين العميل والمريض والكتالوج ومحرك التسعير في تجربة سلسة.

🔗 الحقول المترابطة
العميل → المريض: اختيار العميل يُحدّث تلقائياً قائمة المرضى المرتبطين به
الكتالوج: نوع الخدمة (Transport simple, Transport médicalisé, ECG...)
الاستعجال: Normal أو Urgent — يؤثر على قاعدة التسعير المُطبَّقة
نوع الطاقم: Infirmier · Médecin · Réanimateur — يؤثر على السعر
نقاط A→B: من أين وإلى أين + المسافة بالكيلومتر
التاريخ والوقت: datetime-local مع حل مشكلة timezone
🌙 كشف الوقت الليلي التلقائي
isNightTime(): دالة تكتشف هل الوقت ليلي حسب إعدادات النظام (NIGHT_START_HOUR / NIGHT_END_HOUR)
تفعيل تلقائي: عند اختيار وقت ليلي، المعدِّل NIGHT_SURCHARGE يتفعل تلقائياً مع علامة (auto)
Badge مرئي: 🌙 ليلي / ☀️ نهاري يظهر بجانب حقل التاريخ
قابل للتعطيل: المستخدم يمكنه إلغاء تفعيل المعدِّل الليلي يدوياً
🔧 المعدِّلات (Modifiers)
تحميل ديناميكي: كل المعدِّلات النشطة تُجلب من system_config
checkbox لكل معدِّل: المستخدم يختار ما يريد تطبيقه
عرض النوع: flat_add +100 MAD أو multiplier ×1.5
NIGHT_SURCHARGE: يُعلَّم بـ (auto) إذا فُعِّل تلقائياً

💰 PricingPreview — معاينة السعر الحية

⚡ PricingPreview.tsx
تفاعلي 100%: كل تغيير في الفورم يُعيد حساب السعر فوراً عبر previewPriceAction
مقاوم للـ Race Conditions: عدّاد طلبات (requestCounter) — الاستجابة القديمة تُهمَل
Debounce بـ JSON: JSON.stringify لمقارنة مستقرة للتبعيات — لا إعادة حساب بلا تغيير فعلي
تجربة سلسة: النتيجة السابقة تبقى مرئية (شفافية 50%) أثناء تحميل النتيجة الجديدة
تفصيل كامل: السعر الأساسي + رسوم المسافة + المعدِّلات + TVA + Total TTC

📐 previewPriceSchema — مخطط خفيف للمعاينة

📌 مستقل عن createServiceSchema — لا يتطلب مريض أو عميل أو موقع
📌 حقول أخف: catalogCode · urgency · staffType · distanceKm · selectedModifierIds
📌 scheduledAt اختياري: لأن المعاينة ممكنة قبل تحديد الوقت
📌 الهدف: إرسال أقل بيانات ممكنة لحساب السعر — بدون validation غير ضرورية

🕐 حل مشكلة Timezone

🐛 المشكل: datetime-local يرجع "2026-04-16T22:00" بدون timezone — عند تحويله بـ new Date() يتحول للتوقيت المحلي ويتغير اليوم/الساعة
✅ الحل: نحفظ القيمة كما هي (string محلي) ونحولها لـ ISO فقط عند الإرسال:
const scheduledAtISO = scheduledAt ? new Date(scheduledAt).toISOString() : undefined
✅ كشف الليل: يتم على القيمة المحلية مباشرة — isNightTime(hour, nightStart, nightEnd)

🔄 تدفق البيانات

// 1. Server loads form data loadFormDataAction() → { clients, catalogs, modifiers, nightConfig } // 2. User fills form → ServiceForm builds preview data ServiceForm → { catalogCode, urgency, staffType, distanceKm, modifierIds } // 3. PricingPreview calls server on every change previewPriceAction(previewData) → previewPriceSchema.safeParse() → calculatePrice() // pricing engine — no snapshot → { basePrice, distanceFee, modifiers[], subtotalHt, tva, totalTtc } // 4. User submits → full validation + snapshot createServiceAction(fullData) → createServiceSchema.safeParse() → createService() → calculateAndSavePrice() // pricing engine + snapshot → redirect to /dashboard/services/[id]

🏗️ النمط المعماري

📌 Server Component + Client Component: الصفحة تجلب البيانات، المكون التفاعلي يعرضها
📌 فورم واحد → عرضين: ServiceForm يُنتِج بيانات المعاينة + بيانات الإنشاء
📌 Schema مزدوج: previewPriceSchema (خفيف) + createServiceSchema (كامل)
📌 Race-condition safe: عدّاد + JSON comparison = لا بيانات قديمة تظهر
📌 Timezone safe: التخزين المحلي + التحويل عند الإرسال فقط
📌 Night auto-detect: من إعدادات النظام — nightConfig يُمرَّر من server لـ client
